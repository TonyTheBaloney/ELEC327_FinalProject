Control Map

One shared interface, saved settings per effect

The firmware stores a separate four-pot state for each preset. D1 gates editing, so switching effects does not immediately overwrite saved values.

Control General behavior NeuralSeed behavior
D3 button Cycles through Overdrive, Chorus, Reverb, Phaser, and NeuralSeed. Resets the GRU hidden state when NeuralSeed is selected.
D2 toggle Enables passthrough/bypass by copying audio input directly to output. Bypasses NeuralSeed inference and sends dry audio through.
D1 toggle Enables editing of the current preset's saved pot values. Allows NeuralSeed gain, mix, and level to be updated from the pots.
A1 / Pot 0 Primary parameter for the selected effect. Input gain from 0x to 3x before the GRU model.
A3 / Pot 1 Secondary parameter or depth/mix control. Dry/wet blend between the input signal and neural model output.
A5 / Pot 2 Output, wet level, or feedback-like parameter depending on preset. Final NeuralSeed output level.
A7 / Pot 3 Reserved for expansion in the current firmware. Reserved for a future trained model parameter or tone control.

Signal Path

What happens to the guitar signal

The pedal processes one selected preset at a time inside the Daisy Seed audio callback. At 48 kHz with 48-sample blocks, every block is transformed and written back to the audio output.

01 Guitar input

Analog audio enters the Daisy Seed audio interface.

02 ADC + callback

The firmware receives 48-sample audio blocks at 48 kHz.

03 Selected preset

Only the current effect runs: classic DSP or the NeuralSeed GRU model.

04 Level + blend

Saved pot values shape tone, mix, gain, and output level.

05 Audio output

The processed signal is sent back out to the amp or recorder.

In bypass mode, the callback skips the selected effect and copies left/right input directly to output.

NeuralSeed

Neural amp simulation
on bare metal

A selected GRU10 capture runs entirely on the Cortex-M7 using RTNeural, a header-only inference engine. The Daisy Seed is not training the network or loading models from storage; one trained NeuralSeed-compatible checkpoint is converted into C++ weights and compiled directly into the firmware binary.

How Daisy Seed uses the NeuralSeed model

NeuralSeed provides the trained GRU-style amp/pedal capture and exported weight data. The Daisy firmware embeds those weights in NeuralSeedModelData.h, then RTNeural evaluates the network sample by sample inside the audio callback.

  • Input sample: the current guitar sample is scaled by the input-gain pot.
  • GRU hidden state: ten recurrent values carry short-term memory from previous samples.
  • Dense output: the 10-value hidden state is mapped back to one audio sample.
  • Blend/output: the firmware mixes dry and neural audio, applies level, and writes the output.

Results & Limits

Current project status

The firmware demonstrates a complete real-time effects pedal path, with room for more measurement, enclosure work, and model-selection features.

Working firmware path

Five presets run inside the Daisy Seed audio callback, with per-effect saved pot values and bypass support.

Embedded neural inference

A GRU10 NeuralSeed-compatible checkpoint is compiled into firmware and evaluated sample by sample with RTNeural.

Known limitations

The board uses one compiled NeuralSeed checkpoint at a time; it does not dynamically load or switch model files at runtime.

Future work

Add measured latency, demo recordings, an enclosure photo, A7 tone/model control, and a cleaner model-selection workflow.

Getting Started

Build & flash

Uses a reproducible Nix dev shell — no manual toolchain installation required.

01
Clone the repository 
git clone --branch neuralseed-test https://github.com/TonyTheBaloney/ELEC327_FinalProject
cd ELEC327_FinalProject
02
Pull submodules

Fetch libDaisy, DaisySP, RTNeural, and nested dependencies.

git submodule update --init --recursive
03
Enter the Nix dev shell 
cd AudioProcessing
nix develop --extra-experimental-features "nix-command flakes"
04
Build libraries & firmware 
make -C libDaisy
make -C DaisySP
make
05
Flash to Daisy Seed

Put the board in USB DFU mode, then flash the firmware.

make program-dfu
